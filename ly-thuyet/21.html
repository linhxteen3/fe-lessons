① Tóm tắt lý thuyết

1.1 記憶管理 là gì? — Khái quát

🇯🇵 日本語 記憶管理とは、記憶装置を効率的に利用するための機能。記憶管理はさらに、主記憶装置を管理する「主記憶管理（実記憶管理）」と、仮想記憶を管理する「仮想記憶管理」の2つに分類される。基本情報技術者試験では仮想記憶管理が頻出。
🇻🇳 Tiếng Việt 記憶管理 (Memory management) = chức năng quản lý bộ nhớ một cách hiệu quả. Chia làm 2 loại: 主記憶管理 (実記憶管理) = quản lý bộ nhớ chính, và 仮想記憶管理 = quản lý bộ nhớ ảo. FE hay hỏi về 仮想記憶 / paging.
🇬🇧 English Memory management efficiently allocates memory. It splits into real (main) memory management and virtual memory management. FE exam emphasizes the virtual / paging side.
記憶管理 Memory Management │ ├── 主記憶管理 / 実記憶管理 (Real) ── Slide 第3回 │ 固定区画 / 可変区画 / オーバーレイ / スワッピング ... │ └── 仮想記憶管理 (Virtual) ── Slide 第4回 + Yasashii p.203- セグメンテーション / ページング

🇻🇳 Tiếng Việt Bình thường chương trình "ngủ" trong bộ nhớ phụ (補助記憶装置 HDD/SSD). Khi chạy, OS chuyển vào 主記憶装置 (RAM). Vì RAM dung lượng nhỏ và chứa nhiều chương trình cùng lúc → OS phải quản lý không gian.

1.3 仮想記憶管理 — Virtual memory (Slide 第4回 + Yasashii p.203-208)

🇯🇵 日本語 仮想記憶管理とは、補助記憶装置の一部をあたかも主記憶装置であるかのように扱う方式。プログラムが主記憶の容量を超える大きさの場合でも、仮想記憶上に格納することで実行できる。OSが論理的な記憶領域=仮想記憶を作り、主記憶上の物理的な記憶領域=実記憶に対応付ける。
🇻🇳 Tiếng Việt 仮想記憶 (Virtual memory) = OS tạo một không gian "giả" có vẻ như là RAM rất lớn, thực ra một phần nằm trên HDD/SSD. Nhờ đó, chương trình lớn hơn RAM thật vẫn chạy được. Nguyên tắc:
  • 仮想記憶 (Virtual) = không gian logic OS tạo ra
  • 実記憶 (Real) = RAM vật lý
  • OS ánh xạ 仮想アドレス ↔ 物理アドレス qua bảng tra
🇬🇧 English Virtual memory treats part of secondary storage as if it were main memory. Programs bigger than RAM can run by being stored in virtual memory. OS creates a logical address space (virtual) mapped to the physical address space (real).

動的アドレス変換機構 (DAT — Dynamic Address Translation)

🇻🇳 Tiếng Việt Đơn vị chuyển đổi địa chỉ — MMU (Memory Management Unit) — chuyển 地址 ảo → địa chỉ vật lý trong lúc CPU chạy. Slide gọi là 動的アドレス変換機構.

1.4 ページング方式 — Paging chi tiết

🇻🇳 Tiếng Việt Cả 仮想記憶 (ảo) và 実記憶 (RAM thật) đều được chia thành các ページ (page) kích thước cố định. CPU truy cập theo địa chỉ ảo; OS dùng ページテーブル (page table) để tra ra địa chỉ vật lý tương ứng.

仮想アドレス & 物理アドレス

  • 仮想アドレス (Virtual address): địa chỉ trong không gian ảo (theo thứ tự từ đầu)
  • 物理アドレス (Physical address): địa chỉ thực trong RAM
  • ページテーブル (Page table): bảng tra với 2 cột — cột địa chỉ ảo, cột địa chỉ vật lý tương ứng. Số dòng = số page trong 仮想記憶.

CPU truy cập page — 3 bước (Yasashii)

  1. CPU đọc page A (仮想アドレス = 1) → tra ページテーブル dòng 1
  2. Dòng 1 cho biết 物理アドレス = 2 → CPU truy cập 実記憶 ô số 2
  3. CPU lấy được nội dung page A

ページフォールト・ページイン・ページアウト

用語Khi nào xảy raÝ nghĩa
ページフォールト (Page fault)CPU cần một page nhưng page đó không có trong 実記憶Một loại interrupt báo "trang đang cần không có trong RAM"
ページイン (Page-in)Sau page faultOS đọc page cần từ 補助記憶 → 実記憶
ページアウト (Page-out)Khi 実記憶 đã đầyOS chọn 1 page bỏ ra, ghi từ 実記憶 → 補助記憶 để lấy chỗ trống
🎯 Luồng đầy đủ Yasashii:
仮想記憶へのアクセス → ページテーブル参照 → ページフォールト → 置き換えるページの決定 → ページアウト → ページイン

1.5 ページ置き換えアルゴリズム — Page Replacement

🇻🇳 Tiếng Việt Khi cần page-in nhưng 実記憶 hết chỗ → phải chọn page nào "đẩy ra" (置き換え). Cần chọn page mà khó có khả năng dùng lại, để tránh phải page-in lại ngay. Có 4 thuật toán chính (Yasashii có 3; Slide thêm LIFO):
Thuật toánTiêu chí chọn để đẩy raTên đầy đủ
FIFOPage vào RAM sớm nhất (cũ nhất theo thời điểm nạp)First In First Out
LIFOPage vào RAM mới nhất (gần đây nhất nạp)Last In First Out
LRU ⭐Page lâu nhất không được tham chiếu (最後に参照されてからの経過時間が最も長い)Least Recently Used
LFUPage có số lần tham chiếu ít nhất (参照頻度が最も少ない)Least Frequently Used
📌 Yasashii: 基本情報技術者試験に出題されるのは主に LRU.

1.6 スラッシング — Thrashing

🇯🇵 日本語 スラッシングとは、仮想記憶管理において、割り当てられる実記憶の容量が小さいときに、ページアウトとページインが頻繁に起こる現象である。スラッシングが発生するとコンピュータ全体のスループットが低下する。
🇻🇳 Tiếng Việt スラッシング (Thrashing) = hiện tượng page-in/page-out xảy ra liên tục khi RAM cấp cho chương trình quá nhỏ. Hệ quả: CPU phần lớn thời gian chỉ bận swap page, スループット (throughput) toàn hệ thống giảm mạnh.
🇬🇧 English Thrashing happens when allocated real memory is too small — the system spends most time paging in and out, drastically reducing throughput.
実記憶 ←→ 補助記憶 ↑↓ ↑↓ ↑↓ ページアウト/イン が連続発生 ↓ スループット低下

② Ví dụ minh hoạ

Ví dụ 1: CPU truy cập page A — 3 bước (Yasashii)

Tình huống Page table có:
仮想アドレス物理アドレス
12
2(trống)
CPU cần page có 仮想アドレス = 1 (page A).
  1. CPU đọc 仮想記憶 ô 1 → page A. Tra ページテーブル dòng "1"
  2. Dòng 1: cột 物理アドレス = 2 → CPU truy cập 実記憶 ô 2
  3. Lấy được nội dung của page A ✓

Ví dụ 2: Phân biệt 3 thuật toán bằng bảng tham chiếu (Yasashii)

Đề bài 実記憶 chứa 3 page A, B, C với dữ liệu:
ページ読み込んだ時刻最後に参照した時刻参照回数
A0:000:059
B0:020:034
C0:030:072
Cần page-out 1 page. Hỏi mỗi thuật toán chọn page nào?
  • FIFO: vào sớm nhất → A (0:00)
  • LRU: tham chiếu cuối cùng lâu nhất → B (0:03, cũ nhất so với A=0:05, C=0:07)
  • LFU: số lần tham chiếu ít nhất → C (2 lần)

Ví dụ 3: Mô phỏng FIFO/LIFO/LRU/LFU (Slide)

Đề bài Frame RAM có thể chứa 3 page. Thứ tự CPU truy cập: 1 2 1 1 3 2 3 3 4. Khi page 4 đến, RAM đã đầy {1,2,3} — cần page-out 1 trong số đó.
Thuật toánChọn đẩy raLý do
FIFO1Page 1 vào sớm nhất
LIFO3Page 3 vào muộn nhất (vào thứ 5)
LRU1Sau lần tham chiếu cuối, page 1 đã không được dùng lâu nhất (cuối cùng = bước thứ 4)
LFU2Page 2 được dùng 2 lần, ít nhất (page 1: 3 lần, page 3: 3 lần)

Ví dụ 4: Tình huống thrashing

Tình huống Bạn mở 30 tab Chrome + Photoshop + Video editor cùng lúc trên máy có RAM 4GB. Mỗi ứng dụng cần page mới liên tục, nhưng RAM gần như đầy.
  • Mỗi lần CPU dùng app khác → page fault → page-out của app này, page-in app kia
  • CPU dành 90% thời gian chỉ để swap page → thực sự chạy chương trình rất ít
  • Bạn thấy máy "đứng hình", quạt chạy điên cuồng → đây chính là スラッシング
  • Giải pháp: tăng RAM, đóng bớt app, hoặc tăng kích thước paging file

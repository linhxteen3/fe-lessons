📘 Phần 1 — Tóm tắt lý thuyết

1.1. Phân cấp bộ nhớ (記憶装置の階層)

🇯🇵 コンピュータの記憶装置は左から右へ「処理速度が速く小容量」→「処理速度が遅く大容量」となる：CPU → キャッシュメモリ → 主記憶装置 → 補助記憶装置。
🇻🇳 Bộ nhớ máy tính phân cấp theo tốc độ–dung lượng: CPU → Cache → 主記憶 (RAM) → 補助記憶 (HDD/SSD). Càng gần CPU càng nhanh nhưng dung lượng nhỏ; càng xa CPU càng chậm nhưng dung lượng lớn.
🇬🇧 Memory hierarchy: CPU → cache → main memory → secondary storage. Closer = faster but smaller.
[ CPU ] ⟷ [ キャッシュ ] ⟷ [ 主記憶装置 ] ⟷ [ 補助記憶装置 ] ←─── 高速・小容量 低速・大容量 ───→

1.2. RAM vs ROM (Slide 第6回)

種類特徴例
RAM読み書き可能、電源OFFでデータ消失 (揮発性)DRAM, SRAM
ROM読み出し専用、電源OFFでもデータ保持 (不揮発性)マスクROM, EPROM, EEPROM, フラッシュメモリ

1.3. DRAM vs SRAM (Slide 第6回)

項目DRAMSRAM
原理コンデンサで電荷保持フリップフロップ回路
リフレッシュ必要 (定期的にデータ書き直し)不要
速度遅い速い

🇻🇳 SRAM nhanh và đắt → dùng làm キャッシュメモリ. DRAM rẻ và dung lượng lớn → làm 主記憶 (RAM thông thường). DRAM cần "refresh" định kỳ vì điện tích trong tụ điện rò rỉ.

1.5. ⭐ キャッシュメモリ (Slide 第7回・Yasashii)

🇯🇵 キャッシュメモリとは、CPUと主記憶装置の間にある超高速の記憶装置。CPUは頻繁に使うデータをキャッシュに保存することで、処理の待ち時間を短縮する。
🇻🇳 キャッシュメモリ (cache memory): bộ nhớ siêu nhanh nằm giữa CPU và 主記憶, lưu các dữ liệu hay dùng để CPU không phải truy cập tới 主記憶 chậm. Mục đích: 埋める CPUと主記憶の速度差.
🇬🇧 Cache memory sits between CPU and main memory, storing frequently-used data to reduce wait time.
So sánh đời thực: CPU như đầu bếp giỏi, 主記憶 như kho thực phẩm xa, キャッシュ như giá đồ ăn ngay bên cạnh bếp — bếp lấy từ giá gần nhanh hơn nhiều so với chạy ra kho.

1.6. 多段キャッシュ (Multi-level cache)

🇻🇳 Cache được chia thành nhiều cấp để tăng tốc độ thêm: 内部キャッシュ (1次/2次 cache) nằm trong CPU, 外部キャッシュ (3次 cache) nằm ngoài CPU. Số càng nhỏ → càng nhanh & gần CPU.
[ CPU ] → [ L1 ] → [ L2 ] → [ L3 ] → [ 主記憶 ] 高速・小容量 低速・大容量

1.7. ⭐ ヒット率 (Hit rate)

🇯🇵 必要なデータがキャッシュメモリ上に配置されている確率を「ヒット率」という。一方、キャッシュ上にない確率は「1 − ヒット率」となる。
🇻🇳 ヒット率 (hit rate) = xác suất dữ liệu CPU cần ĐANG NẰM SẴN trong cache. Còn lại "1 − ヒット率" là tỷ lệ phải đi đến 主記憶 (= ミス率/miss rate).
🇬🇧 Hit rate = probability that required data is in cache. Miss rate = 1 − hit rate.

1.8. ⭐ 実効アクセス時間 (Effective Access Time — TRỌNG TÂM)

実効アクセス時間 = キャッシュへのアクセス時間 × ヒット率
+ 主記憶へのアクセス時間 × (1 − ヒット率)
🇻🇳 Đây là công thức quan trọng nhất FE hay hỏi. Yasashii: 「実効アクセス時間を計算する問題がとてもよく出題される」.

1.9. ⭐ ライトスルー方式 vs ライトバック方式 (Write-through / Write-back)

方式動作メリットデメリット
ライトスルーキャッシュと主記憶の両方に同時に書き込む制御がシンプル (いらないデータを追い出す制御不要)毎回主記憶への書き込みが必要 → 遅い
ライトバック普段はキャッシュだけに書き込み、データがキャッシュから追い出される時に主記憶へ書き込む主記憶への書き込み回数が少ない → 速い追い出し制御が複雑
🇻🇳 Điểm khác cốt lõi: thời điểm ghi xuống 主記憶. Write-through ghi ngay lập tức, write-back ghi chỉ khi data bị "đẩy ra" khỏi cache.

1.10. メモリインタリーブ (Memory Interleaving)

🇯🇵 メモリインタリーブとは、CPUから主記憶装置へのアクセスを高速化するために、主記憶装置内部を複数のバンクに分割し、各バンクに並列にアクセスすること。
🇻🇳 メモリインタリーブ: chia 主記憶 thành nhiều "bank" (ngăn), CPU truy cập song song các bank → giảm thời gian chờ. Đây là kỹ thuật tăng tốc khác bên cạnh cache.

🧮 Phần 2 — Ví dụ minh hoạ

Ví dụ 2.1: Tính 実効アクセス時間 (Yasashii nguyên văn)

問題 (Yasashii p.154)
  • キャッシュへのアクセス時間: 20 ナノ秒
  • 主記憶へのアクセス時間: 580 ナノ秒
  • ヒット率: 0.9
実効アクセス時間 = 20 × 0.9 + 580 × (1 − 0.9)
= 18 + 58 = 76 ナノ秒

答え: 76 ns

Ví dụ 2.2: Tính ヒット率 ngược từ 実効アクセス時間 (Yasashii 過去問 4-12 style)

問題 (Yasashii 解答 4-12)

キャッシュ 10 ns, 主記憶 60 ns, 実効アクセス時間 15 ns. Tìm ヒット率 x?

15 = 10x + 60(1 − x)
15 = 10x + 60 − 60x
50x = 45 → x = 0.9

答え: ヒット率 = 0.9 (90%)

Ví dụ 2.3: So sánh ライトスルー vs ライトバック (Yasashii 過去問 4-13)

問題

「データ書込み命令を実行したときに、キャッシュメモリと主記憶の両方を書き換える方式」 và 「キャッシュメモリだけを書き換えておき、主記憶の書き換えはキャッシュメモリから当該データが追い出されるときに行う方式」 — đây là 2 phương thức gì?

答え:

  • 2 phương thức là ライトスルー方式 và ライトバック方式
  • Khác nhau ở thời điểm ghi xuống 主記憶: 即時 vs キャッシュから追い出されるとき

Ví dụ 2.4: Khi ヒット率 thay đổi

問題

キャッシュ 5 ns, 主記憶 100 ns. Tính 実効アクセス時間 với ヒット率 0.95 và 0.99.

ヒット率 0.95: 5×0.95 + 100×0.05 = 4.75 + 5 = 9.75 ns
ヒット率 0.99: 5×0.99 + 100×0.01 = 4.95 + 1 = 5.95 ns

Bài học: Chỉ tăng ヒット率 từ 95% → 99% mà giảm thời gian ~39%. Đó là lý do tối ưu cache rất quan trọng.

Ví dụ 2.5: メモリインタリーブ (Yasashii 過去問 4-14)

問題

「各バンクを並列にアクセスする」 — kỹ thuật này là gì?

答え: メモリインタリーブ. Yasashii: chia 主記憶 thành các バンク, CPU truy cập 並列 → tăng tốc.

📚 Phần 1 — Tóm tắt lý thuyết

0. データ構造 là gì?

🇯🇵 日本語 コンピュータが扱う情報を「データ」といい、主記憶装置に保存される。データの並べ方によって処理効率が大きく変わるため、状況に応じてさまざまな方式で並べる。この主記憶装置上のデータの並べ方を「データ構造」という。
🇻🇳 Tiếng Việt Máy tính xử lý thông tin gọi là "dữ liệu", được lưu trong 主記憶装置 (bộ nhớ chính). Cách sắp xếp dữ liệu trong bộ nhớ chính ảnh hưởng lớn đến hiệu suất xử lý — gọi là データ構造 (cấu trúc dữ liệu).
🇬🇧 English The way data is arranged in main memory is called a data structure. Different structures suit different processing needs.

1. スタック (Stack) — 後入れ先出し / LIFO

🇯🇵 日本語 スタックとは、データを1列に並べて、最後に格納したデータを最初に取り出すデータ構造。イメージは「積み上げられた本」。途中の本を取り出すことはできない。格納=push、取り出し=pop。後入れ先出し(LIFO: Last In First Out)。
🇻🇳 Tiếng Việt Stack là cấu trúc xếp dữ liệu thành 1 hàng, vào sau ra trước — phần tử nhập sau cùng sẽ được lấy ra đầu tiên. Hình dung: chồng sách — chỉ lấy được cuốn trên cùng, không rút giữa được.
  • push X — đẩy dữ liệu X vào đỉnh stack
  • pop — lấy phần tử ở đỉnh ra (không cần ghi tên vì chỉ có 1 lựa chọn)
  • Ứng dụng: lưu lịch sử thao tác → máy tính luôn quay về trạng thái cũ được
🇬🇧 English A stack stores data in LIFO order (Last In, First Out). push adds to the top; pop removes from the top. Like a stack of books — only the top is accessible.

🔁 Stack — thao tác push / pop

push A push B push C pop pop push D [ ] [ ] [C] [B] [A] [D] [ ] [B] [B] [A] [ ] [A] [A] [A] [A] [ ] [ ] [ ] ↑ ↑ ↑ ↑ ↑ ↑ bottom TOP=C TOP=B TOP=A TOP=D Pop 2 lần liên tiếp → trả về C, rồi B (theo thứ tự ngược chiều push)

2. キュー (Queue) — 先入れ先出し / FIFO

🇯🇵 日本語 キュー(待ち行列)とは、最初に格納したデータを最初に取り出すデータ構造。イメージは「レジ待ちの列」。最初に並んだ人が最初に会計をすませる。格納=enqueue、取り出し=dequeue。先入れ先出し(FIFO: First In First Out)。
🇻🇳 Tiếng Việt Queue (hàng đợi / 待ち行列) là cấu trúc vào trước ra trước. Hình dung: hàng người xếp ở quầy thu ngân — ai vào trước được phục vụ trước, không chen ngang.
  • enqueue X — thêm X vào cuối hàng
  • dequeue — lấy phần tử ở đầu hàng ra
  • Ưu điểm: xử lý lệnh theo đúng thứ tự nhập vào
🇬🇧 English A queue uses FIFO order. enqueue adds to the rear; dequeue removes from the front. Like a checkout line — first in, first served.

➡️ Queue — thao tác enqueue / dequeue

enqueue A → enqueue B → enqueue C → dequeue → dequeue → enqueue D front front [A] [A][B] [A][B][C] [B][C] [C] [C][D] rear dequeue 2 lần liên tiếp → trả về A, rồi B (theo đúng thứ tự enqueue)

4. 木構造 (Tree) — Hierarchical structure

🇯🇵 日本語 木構造とは、階層構造を持つデータ構造。植物の木を逆さにしたような形をしている。枝分かれする箇所(○)を節(せつ)といい、節がデータを保持する。最上位の節を根(ね)、末端の節を葉(は)と呼ぶ。
🇻🇳 Tiếng Việt Tree (木構造) là cấu trúc dữ liệu có thứ bậc (階層構造). Hình dạng giống cây cối úp ngược. Các thành phần:
  • 節 (node, nút): điểm phân nhánh, nơi chứa dữ liệu
  • 根 (root, gốc): nút trên cùng (chỉ có 1)
  • 葉 (leaf, lá): nút tận cùng, không có nhánh con
  • 親 (parent) — 子 (child): nút phân nhánh là cha, nút bị phân nhánh đến là con
🇬🇧 English A tree is a hierarchical data structure. The topmost node is the root; terminal nodes are leaves; intermediate ones are simply nodes. Each node may have parent/child relationships.

n分木 (n-ary tree)

🇻🇳 Tiếng Việt Theo Yasashii: nếu mỗi nút cha có tối đa n con, gọi là n分木.
  • 2分木 (nhị phân): mỗi cha có 0~2 con
  • 3分木: mỗi cha có 0~3 con
  • n分木: tổng quát hóa
"分木" thể hiện số lượng con tối đa mà một nút có thể phân nhánh.

2分探索木 (Binary Search Tree, BST)

⭐ Điều kiện 2分探索木 (theo Yasashii):
左の子孫 < 親 < 右の子孫
(Tất cả con cháu bên trái nhỏ hơn nút cha; tất cả con cháu bên phải lớn hơn nút cha)
🇯🇵 日本語 2分木の中でも、親子の値が「左の子孫 < 親 < 右の子孫」の関係になっているものを「2分探索木」という。注意点: 値の制約は「子」ではなく「子孫」である(孫・ひ孫など、さらに下のデータも含む)。
🇻🇳 Tiếng Việt Lưu ý quan trọng: ràng buộc áp dụng cho "子孫" (con cháu), không chỉ cho "子" (con trực tiếp). Tức là cháu, chắt, chút, chít ở phía bên trái cũng phải nhỏ hơn cha; bên phải phải lớn hơn cha.

Lợi ích: Tìm dữ liệu rất nhanh — chỉ cần kiểm tra 階層の数 (số tầng) lần là tìm thấy. Cây thường (không có ràng buộc) phải duyệt từng nút một.

🌳 So sánh: 良い例 vs ダメな例 (theo hình trong Yasashii)

○ 良い例 (BST hợp lệ) (7) ← root / \ (5) (9) / \ (3) (6) Kiểm tra: - 7: trái={5,3,6} đều <7 ✓ phải={9} >7 ✓ - 5: trái={3} <5 ✓, phải={6} >5 ✓ → LIFO: tất cả "子孫" thỏa 左 < 親 < 右
✗ ダメな例 (KHÔNG phải BST) (7) / \ (5) (6) ✗ 右の子孫が / \ 親より小さい (3) (8) ✗ 左の子孫が 親より大きい Sai vì: - 6 ở phải của 7 nhưng 6<7 - 8 ở phải-trái-cháu của 7 nhưng nằm dưới (5), so với ông (7): 8>7 ✗ với BST

💡 Phần 2 — Ví dụ minh hoạ

Ví dụ 1 — Stack: dãy thao tác push/pop

Bài: Thực hiện theo thứ tự: push 1, push 2, push 3, pop, push 4, pop, pop. Hỏi giá trị các lần pop là gì? Sau cùng stack chứa gì?
Bước 1: push 1 → stack: [1] TOP=1 Bước 2: push 2 → stack: [1,2] TOP=2 Bước 3: push 3 → stack: [1,2,3] TOP=3 Bước 4: pop → lấy 3, stack: [1,2] TOP=2 ← pop ra 3 Bước 5: push 4 → stack: [1,2,4] TOP=4 Bước 6: pop → lấy 4, stack: [1,2] TOP=2 ← pop ra 4 Bước 7: pop → lấy 2, stack: [1] TOP=1 ← pop ra 2 Đáp án: pop trả về theo thứ tự = 3, 4, 2. Stack cuối còn: [1]

Ví dụ 2 — Queue: dãy enqueue/dequeue

Bài: Thực hiện: enqueue A, enqueue B, dequeue, enqueue C, enqueue D, dequeue. Hỏi dequeue trả về gì? Cuối cùng queue còn gì?
Bước 1: enqueue A → [A] Bước 2: enqueue B → [A, B] front=A, rear=B Bước 3: dequeue → [B] ← lấy A (vào trước → ra trước) Bước 4: enqueue C → [B, C] Bước 5: enqueue D → [B, C, D] Bước 6: dequeue → [C, D] ← lấy B Đáp án: dequeue trả về = A, rồi B. Queue cuối còn: [C, D]

Ví dụ 3 — Tree: xác định BST hay không

Bài: Cây dưới đây có phải 2分探索木 không?
(10) / \ (5) (15) / \ / \ (3) (7)(12)(20)
Kiểm tra điều kiện 「左の子孫 < 親 < 右の子孫」:
  • Nút 10: trái = {5, 3, 7} đều < 10 ✓ ; phải = {15, 12, 20} đều > 10 ✓
  • Nút 5: trái = {3} < 5 ✓ ; phải = {7} > 5 ✓
  • Nút 15: trái = {12} < 15 ✓ ; phải = {20} > 15 ✓
Kết luận: ✅ Đây là 2分探索木 hợp lệ.

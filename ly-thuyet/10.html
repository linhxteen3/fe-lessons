📖Phần 1 — Tóm tắt Lý thuyết

🔧 アルゴリズム & プログラム
🇯🇵 日本語
アルゴリズムとは、問題を解いたり課題を解決するための手順。プログラムは、そのアルゴリズムをコンピュータが理解できる特定の言語で記述したもの。
🇻🇳 Tiếng Việt
Thuật toán (アルゴリズム) = trình tự các bước để giải quyết vấn đề.
Chương trình (プログラム) = thuật toán được viết bằng ngôn ngữ mà máy tính hiểu được.
Ví von: Thuật toán = công thức nấu cà-ri, Chương trình = công thức đó được viết bằng tiếng Nhật.
🇬🇧 English
An algorithm is a step-by-step procedure to solve a problem. A program is that algorithm written in a language a computer understands.
[アルゴリズム例: カレーを作る] 開始 ↓ 食材を購入する ↓ 食材の下処理をする ↓ 食材を煮込む ↓ 盛り付ける 終了 ← これがアルゴリズム [プログラム] = アルゴリズムをコンピュータ言語で記述

🔁 アルゴリズムの基本構造 — 3つ
🇻🇳 Tiếng Việt
Mọi thuật toán đều được tạo từ 3 cấu trúc cơ bản:
① 順次構造 (Tuần tự)
Thực hiện lần lượt từng bước một, từ trên xuống.
処理1 → 処理2 → 処理3
② 選択構造 (Rẽ nhánh)
Kiểm tra điều kiện (True/False), thực hiện nhánh tương ứng.
if 条件 then 処理A else 処理B
③ 繰返構造 (Lặp)
Lặp lại thực hiện cho đến khi thỏa điều kiện kết thúc.
ループ → 処理 → 条件チェック → ループ (繰り返し)
⚙️ プログラムの4つの性質
性質名カタカナTiếng ViệtÝ nghĩa
再帰リカーシブĐệ quyChương trình tự gọi chính nó trong quá trình chạy
再入可能リエントラントTái nhập đượcNhiều chương trình cùng gọi đồng thời, vẫn chạy đúng
再配置可能リロケータブルTái định vị đượcĐặt ở bất kỳ vị trí nào trong RAM đều chạy được
再使用可能リユーザブルTái sử dụng đượcLoad 1 lần, dùng nhiều lần, không cần load lại
出る! 再帰と再入可能の2つが最頻出。
再帰 = 自分自身を呼び出す | 再入可能 = 複数タスクから同時に呼び出されても正しく動作
🔍 整列・探索アルゴリズム & オーダー
アルゴリズム説明オーダー
ハッシュ法ハッシュ値で格納場所を決定、1回で到達O(1)
2分探索法ソート済みデータを半分に割りながら探索 ※要整列O(log₂n)
線形探索法先頭から順番に1つずつ照合O(n)
クイックソート基準値で小グループ/大グループに分割を繰り返すO(n log n)
バブルソート隣り合う要素を比較・交換を繰り返すO(n²)
ハッシュ法のデメリット: 異なる入力が同じハッシュ値になる「衝突 (Collision)」が起こる可能性がある
2分探索法の2大条件:
① データをあらかじめ整列させておく必要がある
② データを2つに分けながら探索する

1
Luồng đi từ trên xuống dưới (nguyên tắc)

① 絶対値を求めるアルゴリズム (Tính giá trị tuyệt đối)

🇻🇳
Nếu x < 0 → đổi dấu (x = −x). Nếu x ≥ 0 → giữ nguyên. Trả về x.

② 1から5までの累計 (Tính tổng 1+2+3+4+5)

🇻🇳
Khởi tạo X=1, Y=0. Vòng lặp: cộng X vào Y, tăng X lên 1, đến khi X=5.

🇻🇳
Trace Table = bảng theo dõi giá trị biến qua từng bước. Đây là kỹ năng cực kỳ quan trọng trong thi FE để giải flowchart đúng.

📌 Ký hiệu quan trọng trong Flowchart
代入記号: 「←」または「→」は代入を表す。
例: x ← 5 = 変数xに5を代入する (gán giá trị 5 cho biến x)
例: 5 → x = 同じ意味
mod (余り): 割り算の余り。
例: 7 mod 3 = 1 (7÷3 = 2 余り 1)

💡Phần 2 — Ví dụ Minh họa

📚 Ví dụ 1: Stack (スタック) — Bài thi thực tế
問題3-1 A,B,C,D theo thứ tự đến. Dùng 1 Stack. Dãy đầu ra nào khả thi?
Đáp án: C, B, D, A
Thứ tự vào: A → B → C → D Push A: [A] Push B: [A, B] Push C: [A, B, C] Pop → C ← lấy C ra trước (LIFO) Pop → B ← lấy B Push D: [A, D] Pop → D ← lấy D Pop → A ← lấy A cuối cùng Kết quả: C, B, D, A ✓
🚶 Ví dụ 2: Queue (キュー) — Bài thi thực tế
問題3-2 ENQ1, ENQ2, ENQ3, DEQ, ENQ4, ENQ5, DEQ, ENQ6, DEQ, DEQ, DEQ → lấy ra cuối là số nào?
ENQ1: [1] → Queue: 1 ENQ2: [1,2] → Queue: 1,2 ENQ3: [1,2,3] → Queue: 1,2,3 DEQ → lấy 1 → Queue: 2,3 ENQ4: [2,3,4] ENQ5: [2,3,4,5] DEQ → lấy 2 → Queue: 3,4,5 ENQ6: [3,4,5,6] DEQ → lấy 3 → Queue: 4,5,6 DEQ → lấy 4 → Queue: 5,6 DEQ cuối → lấy 5 ← Đáp án: 5 ✓
🔍 Ví dụ 3: 2分探索法 (Binary Search)
🇻🇳
Tìm số "6" trong mảng đã sắp xếp: [1, 2, 3, 4, 5, 6, 7, 8, 9]
Mảng: [1, 2, 3, 4, 5, 6, 7, 8, 9] Bước 1: Phần tử giữa = (1+9)/2 = 5 → giá trị = 5 6 > 5 → Tìm ở nửa phải [6,7,8,9] Bước 2: Phần tử giữa = (6+9)/2 = 7 → giá trị = 7 6 < 7 → Tìm ở nửa trái [6] Bước 3: Phần tử giữa = 6 → TÌM THẤY! ✓ Chỉ cần 3 bước cho 9 phần tử! O(log₂9) ≈ 3.17 ≈ 3 bước
🔁 Ví dụ 4: 再帰的アルゴリズム — 総和 (Tổng đệ quy)
🇻🇳
Tính 1+2+3+4+5 = 15 bằng đệ quy: f(5) → f(5) tự gọi f(4) → f(4) gọi f(3)...
// Định nghĩa hàm đệ quy (Recursive Function) f(x): if x ≤ 1 then return 1 else return x + f(x - 1) // Tính f(5): f(5) = 5 + f(4) = 5 + 4 + f(3) = 5 + 4 + 3 + f(2) = 5 + 4 + 3 + 2 + f(1) = 5 + 4 + 3 + 2 + 1 = 15 ✓
⚡ Ví dụ 5: Sửa lỗi Flowchart (Bài thi thực tế)
問題3-6 Tính tổng 1 đến 100. Flowchart bắt đầu với ① 1→x. Kết quả là 5051 (sai, phải là 5050). Sửa như thế nào?
🇻🇳
Lỗi: Khởi tạo x=1 thay vì x=0, nên đã cộng thêm 1 lần 1 dư.
Sửa: Đổi ① thành 0 → x (khởi tạo x=0)
→ 0 + 1 + 2 + ... + 100 = 5050 ✓
BướcFlowchart sai (x bắt đầu=1)Flowchart đúng (x bắt đầu=0)
①1 → x (SAI)0 → x (ĐÚNG)

✏️Phần 4 — Câu hỏi Ôn tập

Q1. スタックの説明として正しいものはどれか。
(Mô tả đúng về Stack là gì?)
ア. 最初に格納したデータを最初に取り出すデータ構造（先入れ先出し）
イ. 最後に格納したデータを最初に取り出すデータ構造（後入れ先出し）
ウ. データをポインタで数珠つなぎにしたデータ構造
エ. データを階層構造で保持するデータ構造
✅ 答え: イ (LIFO — 後入れ先出し)
スタック = 積み上げた本のイメージ。最後に置いた本から取り出す。
🇻🇳 Stack = LIFO. ア はキュー (FIFO)。ウ は連結リスト。エ は木構造。
Q2. 配列と比較した連結リストの特徴として適切なものはどれか。
(Đặc điểm của Linked List so với Array?)
ア. 要素を参照する場合、ランダムにアクセスできるので処理時間は短い
イ. 要素を更新する場合、ポインタをたどるだけなので処理時間は短い
ウ. 要素を削除する場合、削除した要素から後ろにある全要素を前に移動するので処理時間は長い
エ. 要素を挿入する場合、数個のポインタを書き換えるだけなので処理時間は短い
✅ 答え: エ
連結リストは挿入・削除が速い（ポインタを変更するだけ）。ア・イ・ウはすべて配列の特徴または誤り。
🇻🇳 Danh sách liên kết: chèn/xóa nhanh vì chỉ đổi pointer. Mảng: đọc nhanh vì dùng index.
Q3. クイックソートの処理方法の説明として正しいものはどれか。
(Mô tả đúng về Quick Sort?)
ア. 隣り合ったデータの比較と入替えを繰り返して整列する（バブルソート）
イ. 最小値を選び、先頭と交換する操作を繰り返す（選択ソート）
ウ. 適当な基準値を選び、それより小さなグループと大きなグループに分割し、繰り返す
エ. 整列済みデータの正しい位置に未整列データを挿入する（挿入ソート）
✅ 答え: ウ
クイックソートのキーワード: 「基準値」「2つのグループに分割」「繰り返し」
🇻🇳 Quick Sort: chọn pivot → chia 2 nhóm → lặp lại cho mỗi nhóm. O(n log n)
Q4. 再入可能プログラムの特徴はどれか。
(Đặc điểm của Reentrant Program?)
ア. 主記憶上のどこのアドレスに配置しても実行できる（リロケータブル）
イ. 手続の内部から自分自身を呼び出すことができる（リカーシブ）
ウ. 主記憶に読み込まれたら再ロードなしに繰り返し利用できる（リユーザブル）
エ. 複数のタスクからの呼び出しに対して、並行して実行されても正しい結果を返す
✅ 答え: エ
再入可能（リエントラント）= 複数タスクから同時に呼ばれても正しく動作。
🇻🇳 Reentrant = nhiều task gọi đồng thời vẫn đúng. ア=Relocatable, イ=Recursive, ウ=Reusable.
Q5. 流れ図記号で ◇（菱形）は何を表すか。
(Hình thoi ◇ trong flowchart biểu thị điều gì?)
ア. 端子（流れ図の開始と終了）
イ. 処理（計算や代入）
ウ. 判断（条件分岐 — Yes/No で処理を分ける）
エ. ループ端（繰り返し処理の開始と終了）
✅ 答え: ウ
◇ = 判断（Decision）。出口に Yes/No を書く。ア=⬭ 端子、イ=▭ 処理。
🇻🇳 Hình thoi = Điều kiện rẽ nhánh (Yes/No). Hình bầu dục = Bắt đầu/Kết thúc.
Q6. Ajax の説明として正しいものはどれか。
(Mô tả đúng về Ajax?)
ア. JavaScriptの非同期通信を使い、画面全体の遷移を伴わずに動的なUIを実現する技術
イ. HTML文書の文字サイズや色などのデザインを指定する言語（CSS）
ウ. Webページの更新情報を配信するための文書フォーマット（RSS）
エ. 利用者が任意のタグを定義できるマークアップ言語（XML）
✅ 答え: ア
Ajaxキーワード:「非同期通信」「画面遷移なし」「動的UI」
🇻🇳 Ajax = 非同期 (Asynchronous) JS + XML. Cập nhật trang không reload. Google Maps là ví dụ điển hình.
Q7. XMLの特徴として最も適切なものはどれか。
(Đặc điểm đúng nhất của XML?)
ア. HTMLと同様にタグの使い方が固定されており、独自タグは定義できない
イ. ネットワーク経由のデータ交換を容易にするため、任意のタグを定義することができる
ウ. Webページのデザイン（文字色・大きさ）を指定するための言語
エ. サーバ上で動作し、動的なWebページを生成するプログラム（サーブレット）
